## Visual description
- **Components** (top → bottom, PRISMA "Identification of studies via databases and registers"):
  - Identification: "Records identified from Pubmed, Embase, Cochrane, WOS: Databases (n = 179 + 311 + 256 + 195)" → box "Records removed before screening: Duplicate records removed (n = 219); Records marked as ineligible by automation tools (n = 35 + 41 + 28 + 17); Records removed for other reasons (n = 0)".
  - Screening: "Records screened (n = 601)" → excluded box "Not relevant to topic (n = 413); Wrong population (n = 47e (e.g., editorial, letter) (n = 35))".
  - "Reports assessed for eligibility (n = 106)" → excluded box "Ineligible study design (Non-RCT): n = 41; Ineligible population/disease: n = 25; Duplicate data overlap: n = 13; Ineligible intervention (TCM): n = 9".
  - Included: "Studies included in review (n = 18)".
- **Connections**: standard top-down PRISMA flow with right-pointing arrows to exclusion boxes at each stage.
- **Annotations**: colored stage bands on the left (Identification / Screening / Included).
- **What it conveys**: the funnel from initial database records to the final 18 included studies (24 independent datasets after de-overlapping).
